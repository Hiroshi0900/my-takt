# Policy: docs-work-only-e2e-map

## 目的

E2E ケース台帳（Coverage Map）初版を、既存コードと衝突しにくいドキュメント作業として実施する。

## スコープ制約（最重要）

- **編集対象は `work/*.md` のみ**
- `src/test/e2e/**` は **読み取り専用**
- アプリコード（`src/**`）の修正は禁止
- Playwright 設定 / Makefile / CI workflow の変更は禁止
- E2E テストコードの修正・追加・削除は禁止

## MUST（必須）

- 分類基準を明記する（ファイル名ベースの粗分類を含む場合もその旨を明記）
- `P0/P1/P2` の定義を明記する
- `mock / local-real-api / stg` の定義を明記する
- ケースID、目的、受け入れ条件（AC）、次アクションの粒度で記述する
- `spweb` 系や特殊用途 spec の扱いを分離候補として明記する
- 初版の限界（中身再確認が必要な箇所）を明記する

## MUST NOT（禁止）

- テスト実行（`npx playwright`, `make test-e2e` 等）
- コード修正の提案を「今回の実施内容」として混ぜる
- 曖昧な優先度付け（理由なしの P0 指定）
- 台帳に機密情報や実行環境固有の秘匿値を記載する

## 品質基準

- 後続の TDD 的 E2E 追加・修正の入口として使える粒度であること
- 完了条件が文書から確認できること（ファイル存在、セクション、ケースID）
- 将来の精査前提でも、初版として破綻しない分類構造であること
