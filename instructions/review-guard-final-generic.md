# Instruction: review-guard-final-generic

## 目的

実装されたコードを全観点からレビューし、未解消の重要課題が残った状態で完了しないように、完了ゲートと検証証跡を厳格に確認する。

## チェック手順

### 観点 1: Clean Architecture

```
[ ] domain層にインフラ依存 import がないか
[ ] ハンドラー/コントローラが domain の危険なコンストラクタを直接使っていないか
[ ] リポジトリI/Fが内側の層で定義されているか
[ ] ユースケース実装が adapter/application 層にあるか
```

### 観点 2: DDDパターン

```
[ ] 値オブジェクトの生成・検証ルールが明確か
[ ] ルール表現（Specification/Validator）が一貫しているか
[ ] 命名がドメイン用語に沿っているか
[ ] 曖昧なサフィックスがないか
```

### 観点 3: エラー処理

```
[ ] エラーを握りつぶしていないか
[ ] エラーが適切にラップされているか
[ ] 境界変換（HTTP等）にマッピング漏れがないか
```

### 観点 4: テスト

```
[ ] 正常系があるか
[ ] 入力バリデーション異常系があるか
[ ] 依存エラー異常系があるか
[ ] 境界値テストがあるか
```

### 観点 5: 実行ベース品質ゲート（PR前・必須）

1. リポジトリ内の実行コマンドを特定する（優先順）
   - `make lint` / `make test` / `make build`
   - `task lint` / `task test` / `task build`
   - `just lint` / `just test` / `just build`
   - `npm|pnpm|yarn|bun run lint|test|build`
   - Go系の直接実行（例: `golangci-lint run`, `go test ./...`, `go build ./...`）
2. `lint` / `test` は必須:
   - どちらかを特定できない場合は **MUST FIX**
3. `build` は該当する場合に実行する（最低: プロジェクト標準の build コマンド）
4. コマンド名・終了コード・失敗要点をレポートに記録する

### 観点 6: 要件トレーサビリティ（必須）

```
[ ] 指示書/Specの全要件IDを列挙したか
[ ] 各要件IDに対して実装箇所（ファイル:行）を紐付けたか
[ ] 各要件IDに対してテストケース（テスト名/ファイル）を1:1で紐付けたか
[ ] 紐付け漏れを0件にしたか
```

### 観点 7: 契約整合（OpenAPI）（必須）

```
[ ] OpenAPI定義の対象エンドポイント/操作を特定したか
[ ] 成功ステータスとレスポンス型の整合を確認したか
[ ] 失敗ステータスとエラー応答型の整合を確認したか
[ ] 差分（仕様と実装の不一致）を0件にしたか
```

### 観点 8: 再レビュー（必須）

```
[ ] 前回レビューの指摘一覧を取得したか
[ ] 各指摘に対して「解消/未解消」を判定したか
[ ] 解消エビデンス（コミット差分・再実行結果）を記録したか
[ ] 未解消0件を確認したか
```

## 判定ルール（追加）

- `Critical` / `High` の未解消指摘が1件でもある場合は **必ず MUST FIX（COMPLETE不可）**
- `Warning` は「許容理由」が明記されない限り **COMPLETE不可**
- `lint` / `test` が未特定・未実行・失敗の場合は **必ず MUST FIX（COMPLETE不可）**
- OpenAPI差分が1件でもある場合は **必ず MUST FIX（COMPLETE不可）**
- 要件トレーサビリティ（実装箇所 + テスト）の紐付け漏れがある場合は **必ず MUST FIX（COMPLETE不可）**
- 最低限「プロジェクト標準の build コマンド」と対象テストの結果証跡（コマンド・結果・失敗時の対応方針）がない場合は **COMPLETE不可**
- 再レビューの未解消指摘がある場合は **COMPLETE不可**

## 問題の分類

```
🔴 Critical（MUST FIX）: 完了判定をブロックする問題（契約不整合、実行ゲート失敗、重大設計違反）
🟠 High（MUST FIX）: 仕様未充足・重要な品質欠陥・重要テスト欠落
🟡 Warning（理由必須）: 直ちに致命ではないが放置リスクがある問題
🟢 Info（任意）: 改善提案
```

## 出力

`guard-report.md` フォーマットでレポートを作成する。
