PRコメントに対する修正の品質を検証せよ。

**やること:**
1. `comment-triage.md` の「対応必須」コメントを読み込む
2. 各修正が指摘内容を正しく解消しているか確認する:
   - 修正対象のファイルとdiffを確認する
   - 指摘の意図に沿った修正になっているか判定する
3. 品質ゲートを実行する:
   - lint コマンドを実行し成功を確認する
   - test コマンドを実行し成功を確認する
   - build コマンドを実行し成功を確認する（該当する場合）
4. リグレッション確認:
   - 修正が既存テストに影響していないか確認する

**判定ルール:**
- lint / test のいずれかが失敗している場合は「検証不合格」
- 指摘が正しく解消されていない修正がある場合は「検証不合格」
- すべて問題なければ「検証合格」
