# Instruction: refactor-packages-implement

移行計画に従って、パッケージ/モジュール再編を安全に実装せよ。

## 実装原則

- 一回の差分では 1 バッチ分だけを完了する
- ファイル移動後の参照更新を漏らさない
- public surface の変更は最小限に留める
- 既存挙動を壊さないため、必要なら暫定的な橋渡しを置く

## 実装手順

1. 今回のバッチ対象を `refactor-plan.md` から特定する
2. 必要な移動、分割、import更新を行う
3. 関連テストを更新または追加する
4. `lint`、`test`、`build` を実行し結果を記録する
5. 次バッチへ進むべきか、レビューに回すべきかを判断する

## 完了条件

- 今回バッチの構造変更が反映されている
- 参照切れがない
- `lint` / `test` / `build` の結果が確認されている
- 次バッチに送る残件が明確
