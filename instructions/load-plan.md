既存の superpowers 形式の計画ファイルを読み込み、後続 movement に進むための状態を確認する。

**やること:**

1. `docs/superpowers/plans/` 配下の `.md` ファイルを `ls -t` で列挙
2. ユーザータスク指示に feature 名が明示されていれば `{YYYY-MM-DD}-{feature}.md` の形式で絞り込む
3. 最新の計画ファイルを1つ特定
4. 計画ファイルを読み、以下を確認:
   - `[PLAN: ready]` タグが先頭にあるか
   - 未チェックタスク (`- [ ]`) が1件以上あるか
   - プレースホルダ `{...}` がタスク内に残っていないか
5. Previous Response があり `00-plan.md` 相当の前回バッチ情報が残っている場合は参照する

**必須出力:**

```markdown
# 計画ロード結果

## 計画ファイル
- path: `docs/superpowers/plans/{file}`
- ready タグ: あり | なし
- 未チェックタスク数: {N}

## 進捗
- 完了タスク: {N}
- 未完了タスク: {N}

## 判定
- **結果**: 計画有効 / 計画不足・不備
- **根拠**: {1-2文}
```

**判定ルール:**

- ready タグあり、未チェックタスク>0、プレースホルダなし → `計画有効`
- いずれか欠ける → `計画不足・不備` (呼び出し元の sdd-impl が終了する)
