## 監督フェーズ（完了ゲート強化）

**実行ベース品質ゲート（必須）**
- `lint` / `test` を特定し、両方を実行して成功を確認する
- 最低限「プロジェクト標準の build コマンド」と対象テストの実行結果（コマンド・結果・失敗時の対応方針）を `supervisor-validation.md` に記録する（例: Go は `go build ./...`, Node は `npm run build`）
- `lint` / `test` 未特定・未実行・失敗、および build失敗は完了判定不可

**完了ゲート（必須）**
- Critical / High 未解消指摘が 0 件
- Warning は許容理由が明記されたもの以外 0 件

**要件トレーサビリティ（必須）**
- 指示書/Specの要件ごとに、実装箇所（ファイル:行）とテストケース（ファイル:テスト名）を1:1で紐付ける
- 紐付け漏れがある場合は完了判定不可

**OpenAPI契約整合（必須）**
- OpenAPI定義との整合（成功/失敗ステータス、レスポンス型）を確認する
- 差分が残る場合は完了判定不可

**再レビュー（必須）**
- 前回指摘の解消状況をチェックリストで確認する（解消/未解消/許容）
- 未解消が残る場合は完了判定不可
