# Instruction: investigate-generic

## 目的

修正・拡張対象のコードを調査し、現状の事実を把握する。推測ではなくコードを根拠に判断する。

## 手順

### Step 1: 対象の特定

- 対象ドメイン/モジュール
- 対象の操作（ユースケース、値オブジェクト、リポジトリ等）
- 症状（バグ）または追加したい振る舞い（機能）

### Step 2: ファイル構造の把握

```
1. 対象モジュールの全ファイルを列挙（Glob）
2. 各層（domain/usecase/repository/interface_adapter 等）の配置を確認
3. 既存の類似機能の場所を特定
```

### Step 3: 対象コードの精読

```
1. 関数シグネチャと実装の両方を読む
2. エラー処理フローを追う
3. 依存関係（import / DI）を確認する
```

### Step 4: 参照関係の調査

```
1. 型・関数の使用箇所を検索
2. テストファイルを確認
3. composition root（main.go / wire.go 等）で wiring を確認
```

### Step 5: 類似コードの調査

```
1. 同じ層の類似実装を2-3件読む
2. プロジェクト固有のパターンを抽出する
3. 今回従うべきパターンを決める
```

## 出力

`investigation-report.md` フォーマットで調査レポートを作成する。
