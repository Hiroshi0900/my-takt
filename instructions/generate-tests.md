# Instruction: generate-tests-generic

## 目的

実装したコードのテストを生成する。ユースケーステスト・値オブジェクトテスト・必要に応じた統合テストを作成する。

## 手順

### Step 1: 既存テストパターンの確認

```
1. 同じモジュールの既存テストを読む
2. モック生成方法、命名規則、アサーション方針を確認する
3. テストデータ生成ヘルパーの再利用方針を確認する
```

### Step 2: ユースケーステスト

最低限、以下を用意する：

```
1. 正常系
2. 入力バリデーション異常系
3. リポジトリ/依存エラー異常系
4. ドメインルール違反
5. 境界値
```

### Step 3: モック設定

- 依存をモック化し、期待呼び出しと戻り値を明示する
- 1テストケースにつき何を保証したいかを明確にする

### Step 4: 値オブジェクトテスト

- 有効値
- 無効値
- 境界値
- 危険なコンストラクタ（panic版）がある場合は panic テスト

### Step 5: 統合テスト（必要な場合）

- 実インフラ依存が必要な検証のみ統合テストに分離する
- 後片付け（データ cleanup）を必ず行う

## テストファイル配置

```
domain/xxx_test.go
interface_adapter/usecase/xxx_test.go
internal/test/integration/... (必要な場合)
```
