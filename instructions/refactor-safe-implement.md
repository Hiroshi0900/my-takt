# Instruction: refactor-safe-implement

段階移行計画に従って、安全性を優先しながらリファクタリングを実装せよ。

## 実装原則

- 今回のバッチ以外に手を広げない
- 互換維持のための橋渡しコードは明示的に残す
- テストや監視の追加を先行させることを許容する
- 危険な一括 rename / move を避ける

## 実装手順

1. 今回バッチの完了条件を確認する
2. 互換戦略に従って変更を実装する
3. 必要なテスト、移行補助、フォールバックを追加する
4. `lint` / `test` / `build` を実行する
5. 次バッチへ進めるか判定する

## 完了条件

- 今回のバッチ範囲だけが確実に完了している
- 既存利用者の破壊を避けるための配慮が入っている
- 実行結果が記録されている
