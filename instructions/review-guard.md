# Instruction: review-guard-generic

## 目的

実装されたコードを全観点からレビューし、Clean Architecture違反・DDD逸脱・エラー処理の欠落・テスト不足を検出する。

## チェック手順

### 観点 1: Clean Architecture

```
[ ] domain層にインフラ依存 import がないか
[ ] ハンドラー/コントローラが domain の危険なコンストラクタを直接使っていないか
[ ] リポジトリI/Fが内側の層で定義されているか
[ ] ユースケース実装が adapter/application 層にあるか
```

### 観点 2: DDDパターン

```
[ ] 値オブジェクトの生成・検証ルールが明確か
[ ] ルール表現（Specification/Validator）が一貫しているか
[ ] 命名がドメイン用語に沿っているか
[ ] 曖昧なサフィックスがないか
```

### 観点 3: エラー処理

```
[ ] エラーを握りつぶしていないか
[ ] エラーが適切にラップされているか
[ ] 境界変換（HTTP等）にマッピング漏れがないか
```

### 観点 4: テスト

```
[ ] 正常系があるか
[ ] 入力バリデーション異常系があるか
[ ] 依存エラー異常系があるか
[ ] 境界値テストがあるか
```

## 問題の分類

```
🔴 重大（MUST FIX）: アーキテクチャ違反、エラー握りつぶし
🟡 中程度（SHOULD FIX）: DDD逸脱、テスト不足
🟢 軽微（NICE TO HAVE）: 命名、コメント
```

## 出力

`guard-report.md` フォーマットでレポートを作成する。
