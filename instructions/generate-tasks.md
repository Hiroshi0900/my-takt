要件と設計に基づき、実装タスクリストを生成せよ。

**注意:** 要件と設計が `openspec/specs/{feature}/` に存在しない場合はABORTする。

**やること:**
1. タスクから対象feature名を特定する
2. `openspec/specs/{feature}/requirements.md` を読み込む
3. `openspec/specs/{feature}/design.md` を読み込む
4. `openspec/specs/{feature}/research.md` があれば読み込む
5. 設計コンポーネントを実装タスクに分解する:
   - 全要件がタスクにマッピングされることを確認
   - 設計コンポーネントが全てカバーされることを確認
   - タスク間の依存関係を分析する
6. `PRバッチ計画` を作成する（小さめPR単位）
   - `PR-1`, `PR-2`, ... の識別子で区切る
   - 各PRバッチに「目的」「対象タスクID」「完了条件」を記載する
   - 原則 1バッチ = 1目的、最初に最小縦スライスを置く
   - CI統合や大規模移行は将来バッチへ分離する
7. 並列実行可能なタスクに `(P)` マーカーを付与する
8. 成果物を保存する

**成果物の保存先:**
- `openspec/specs/{feature}/tasks.md`

**必須出力**

`PRバッチ計画` + チェックボックス形式のタスクリスト。各タスクに:
- 自然言語での振る舞い記述（ファイルパス・関数名禁止）
- 詳細項目（箇条書き）
- 要件マッピング（`_Requirements: X.X, Y.Y_`）
- 並列可能なら `(P)` マーカー

```markdown
## PRバッチ計画

### PR-1: {目的}
- 目的: {1つの目的}
- 対象タスク: {例: 1.1, 1.2, 2.1}
- 完了条件: {PRとしてレビュー可能な完了状態}

### PR-2: {目的}
- 目的: {1つの目的}
- 対象タスク: {例: 2.2, 2.3}
- 完了条件: {完了状態}

- [ ] 1. メジャータスク概要
- [ ] 1.1 (P) サブタスク記述
  - 詳細項目1
  - 詳細項目2
  - _Requirements: 1.1, 1.2_
```
