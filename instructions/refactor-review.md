# Instruction: refactor-review

リファクタリング差分をレビューし、構造問題が解消されたかを判定せよ。

## レビュー観点

1. 元の構造問題が実際に軽減または解消されたか
2. 新しい循環依存や責務混在が増えていないか
3. 名前と配置が境界意図を表現できているか
4. 一度の差分として大きすぎないか
5. `lint` / `test` / `build` の証跡があるか

## 判定

- `承認`: 主要問題が解消され、次へ進める
- `要修正`: 重大問題が残る、または新しい問題を導入した
- `要再設計`: 実装ではなく計画の見直しが必要

## 出力要求

`refactor-review.md` に、重要度付きの指摘と判定理由をまとめること。
