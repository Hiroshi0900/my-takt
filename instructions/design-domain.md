# Instruction: design-domain-generic

## 目的

調査結果を元に、DDDドメインモデルを設計する。値オブジェクト・エンティティ・集約・エラー型・リポジトリI/F を必要最小限で定義する。

## 手順

### Step 1: 既存パターンの確認

```
1. 同じモジュールの既存値オブジェクトを2-3件読む
2. 既存のエンティティ/集約を確認する
3. 既存エラー型・エラー変換方法を確認する
```

### Step 2: 値オブジェクトの設計

```
1. 有効値の条件（空文字不可、範囲制約等）を明文化
2. ルール表現（Specification/Validator）を設計
3. 安全コンストラクタ（SafeNew* 等）を設計
4. ファイル名と配置を決める（1型1ファイル原則）
```

### Step 3: エンティティ・集約の設計（必要な場合）

```
1. 集約境界を決める
2. 状態と状態遷移ルールを整理する
3. コマンドメソッドの戻り値（新状態 / event / error）を決める
```

### Step 4: エラー型の設計

```
1. 標準エラー型で対応可能か確認
2. 新規エラーが必要なら、ビジネス意味のある命名にする
3. 境界（HTTP等）へのマッピング方針を決める
```

### Step 5: リポジトリI/Fの設計

```
1. コマンドリポジトリ（永続化）
2. クエリリポジトリ（読み取り、必要なら）
3. モック生成方針（go:generate 等）を設計に含める
```

## 出力

`design-report.md` フォーマットで設計レポートを作成する。
